VC_5_CyDorm

/registration

Used to create new users. New users enter name, net-id, and their user role. On submission, the user is sent to the server and added to MySQL table. The role in role-table is linked to user.

/login

User enters their username and password. If a login is successful, it will redirect to /navbar and load in the user's data.

/navbar

This is a restricted page; you must be successfully logged in to access it. This page is where all core functionality occurs. If you are a user, you will be able to add or modify groceries and send disputes to your TA.

/ca/caView

This is a restricted page; you must be successfully logged in as a CA to use it. This view will show you all disputes sent by students in your sublocation (which is like a hall).

/admin/addResidency

This is a restricted page; you must be successfully logged in as an admin to access it. This page allows you to add residential locations to the database. Admins will enter a location,sublocation, and address. On submission, its added to database.

/admin/assignCA

This is a restricted page; you must be successfully logged in as an admin to access it. This page allows administrators to assign CAs to certain sublocations.

/admin/setUserToResidency

This is a restricted page; you must be successfully logged in as an admin to access it. This page allows administrators to assign users to certain addresses.
